# 人類、LLM、程式三方比較

「人類直覺」、「LLM 概率推理」與「程式確定性邏輯」這三者之間的本質差異。

## 三者的核心特徵

| 比較維度 | 人類（生物智能） | LLM（機率模型） | 程式（確定性邏輯） |
|---|---|---|---|
| **本質** | | | |
| 核心組成 | 神經系統 + 價值驅動 | token + 機率預測 | 確定狀態機 + 演算法 |
| 可解釋性 | 可用語言解釋 | 黑箱作業 | 100%可理解 |
| 語言理解 | ✅ 深層語境、情感、弦外之音 | ✅ 非結構化文本、翻譯、摘要 | ❌ 只能處理格式化輸入 |
| 創意發想 | ✅ 真正原創、突破性思維 | ✅ 跨域重組、靈感激發 | ❌ 完全無法 |
| 精確計算 | ❌ 容易出錯、速度慢 | ❌ 常有幻覺，不可靠 | ✅ 零誤差、毫秒級執行 |
| 重複執行 | ❌ 疲勞、品質不穩定 | ⚠️ 可大量處理，但有隨機性 | ✅ 億次執行結果完全一致 |
| 模糊判斷 | ✅ 直覺、常識、情境推理 | ✅ 開放性問題、彈性應對 | ❌ 規則外即失效 |
| 樣本學習 | ✅ 看一兩個例子即可類推 | ⚠️ 少樣本可用，但需預訓練 | ❌ 需明確定義所有規則 |
| 物理行動 | ✅ 複雜操作、突發應變 | ❌ 無具身能力(未來可行) | ⚠️ 工業機器人可，但靠規則 |
| 失敗模式 | ⚠️ 認知偏誤／情緒影響 | ❌ 幻覺 — 流暢但不準確 | ⚠️ 崩潰 — 遇到例外直接錯誤 |
| 可靠性 | ⚠️ 狀態、情緒影響品質 | ⚠️ 相同問題可能給不同答案 | ✅ 100% 可重現 |
| 執行速度 | ❌ I/O 慢、算力低 | ⚠️ API 呼叫有延遲與成本 | ✅ 極速，幾近零成本 |
| 責任歸屬 | ✅ 可承擔法律與道德責任 | ❌ 無法承擔責任 | ❌ 無法承擔責任 |
| **Agentic 系統定位** | | | |
| 角色 | 董事長：設定目標與把關 | 專案經理：理解與拆解任務 | 執行者：精確計算與自動化 |
| 需求適配 | 風險價值判斷? | 模糊語意? | 100%正確執行? |
